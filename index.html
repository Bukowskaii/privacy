Privacy Policy — Whale Shark Watch Face

Effective 21 July 2026 · Contact: me@bukowskaii.com

Summary

Whale Shark Watch Face does not collect, store, transmit or share any personal data. It contains no application code, no network access and no analytics.

Why this policy is short

A Watch Face Format face is a resource-only package. It is a set of images and an XML description of how to draw them; the manifest declares android:hasCode="false", and the published bundle contains no executable code at all. The watch's own system software reads that description and renders the dial.

There is therefore no component of this app that is capable of collecting information, opening a network connection, or sending anything anywhere.

Complications

The face can display complications — the small readings in the dial's slots, such as heart rate, step count, date or weather.

That data is produced by other applications on the watch and supplied to the watch face system by the operating system, at your request, when you choose a provider for a slot. The face describes where on the dial to draw the value. It does not receive that data into any code of its own, and does not retain, copy or transmit it. Nothing is written to storage, and nothing leaves the watch.

To stop a reading being displayed, set that complication slot to empty in the watch face editor.

Permissions

The package declares the following permissions, which allow the system to supply complication data to the face:

PermissionPurpose
RECEIVE_COMPLICATION_DATAAllows the system to deliver complication values for display
BODY_SENSORSAllows heart-rate complications to be shown
BODY_SENSORS_BACKGROUNDAllows a heart-rate complication to update while the display is dimmed
ACTIVITY_RECOGNITIONAllows step-count and activity complications to be shown

These permissions govern what the system may pass to the face for rendering. The face has no code able to read a sensor directly.

Children

This app is not directed at children and contains nothing directed at children.
